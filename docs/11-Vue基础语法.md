# 11 Vue 基础语法

## 适用范围

- 处理 uni-app 中 Vue 模板、响应式数据、事件与列表渲染等基础问题时使用。

## 规则

- 页面模板优先使用标准 Vue 指令：`v-if`、`v-show`、`v-for`、`v-model`、`v-bind`、`v-on`。
- 列表渲染必须提供稳定 `key`，禁止使用会频繁变化的临时值。
- 事件处理参数与回调职责保持清晰，避免在模板中堆复杂表达式。
- 双向绑定仅用于表单输入等场景，复杂业务状态优先显式更新。
- 计算属性用于派生状态，副作用逻辑不要放在计算属性中。
- 侦听器（watch）用于响应状态变化触发副作用，避免与模板渲染职责混用。
- Vue2 风格页面（Options API）中方法统一写在 `methods`，避免 `this` 指向混乱。

## 示例

- 条件渲染：`v-if` 控制加载态与空态切换。
- 列表渲染：`v-for="item in list" :key="item.id"`。
- 双向绑定：`v-model` 绑定输入框，再在提交动作统一校验。
- 计算属性：根据原始数据派生展示文案，不直接修改源状态。

## 反例

- `v-for` 使用索引或随机值作为 `key`，导致渲染抖动与状态错位。
- 在模板里写大量复杂三元表达式，降低可读性与可维护性。
- 计算属性内发请求或改状态，造成难以追踪的副作用。
- Vue2 页面方法使用箭头函数，`this` 指向异常。

## 检查项

- 模板是否仅保留展示逻辑，复杂处理是否下沉到脚本层。
- 列表 `key` 是否稳定且业务语义明确。
- 计算属性与侦听器是否职责分离。
- Vue2 风格页面中 `methods` 与 `this` 使用是否正确。
