# Uni-app SliceLab Skill

## 执行目标

- 基于切片文档快速定位规则。
- 在改动前明确约束与边界，避免风格漂移。
- 输出简洁、模块化、可维护的实现方案。

## 快速流程

1. 先读 `docs/00-index.md`，确定要查的主题切片。
2. 按任务类型读取对应切片（只读必要文件）。
3. 提炼“必须遵守”的规则并形成执行清单。
4. 先给用户说明思路，再进行代码或文档改动。
5. 改动后按切片内“检查项”自检。

## 切片选择规则

- 目录规划或新模块落地：`docs/01-项目结构.md`
- 页面新增与导航流程：`docs/02-路由与页面.md`
- Pinia/缓存/登录态：`docs/03-状态与缓存.md`
- 请求层、鉴权、错误处理：`docs/04-请求与拦截器.md`
- 组件拆分与命名、样式层级：`docs/05-组件与样式规范.md`
- H5/App/小程序差异处理：`docs/06-多端差异处理.md`
- 构建、环境与发版：`docs/07-打包发布.md`
- 常见异常排查：`docs/08-常见问题.md`
- App 原生插件与鸿蒙边界：`docs/09-App原生插件与能力.md`
- 页面样式与布局兼容：`docs/10-页面样式与布局.md`
- Vue 指令与响应式基础：`docs/11-Vue基础语法.md`
- 业务实体与状态规则：`docs/business/01-业务模型模板.md`（或对应模块文件）
- 接口入参出参与错误码：`docs/business/02-接口契约模板.md`（或对应模块文件）
- 页面与业务分支流程：`docs/business/03-业务流程模板.md`（或对应模块文件）
- 提测与上线验收清单：`docs/business/04-业务验收模板.md`（或对应模块文件）

## 输出约束

- 保持简洁代码风格，避免过深层级和冗余封装。
- 优先组件化/模块化，不把大段逻辑写在单文件。
- 方法、模块写简短注释，说明职责与输入输出。
- 命名与既有风格一致，避免引入新命名体系。

## 维护要求

- 新增规则时，先更新对应切片，再补 `docs/00-index.md` 索引。
- 同一规则只放一个切片，避免重复与冲突。
- 每条关键规则都要包含“示例 + 反例 + 检查项”。
